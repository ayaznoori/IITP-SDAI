# 1️ What is Problem Solving in DSA?

Problem solving in Data Structures and Algorithms (DSA) means:

* Understanding the problem clearly
* Breaking it into smaller steps
* Choosing the correct technique
* Writing an efficient solution

In coding interviews and real software work, solving a problem is not only about getting the correct answer — it is also about solving it efficiently.

---

# 2️ Why Searching and Sorting Matter

Searching and sorting are two of the most important techniques in programming.

### Searching

Searching helps us find whether an element exists, where it exists, or how it should be used.

Examples:

* Find a student’s roll number in a list
* Check whether a product ID exists
* Search for a target number in an array

### Sorting

Sorting helps us arrange data in a useful order.

Examples:

* Arrange marks in ascending order
* Sort names alphabetically
* Order tasks by priority

Many difficult problems become easier after sorting.

---

# 3️ Why Recursion is Useful

Recursion means a function solves a problem by calling itself on a smaller part of the same problem.

Example ideas:

* Searching in a smaller half of an array
* Breaking an array into smaller parts
* Solving divide-and-conquer problems

Recursion is often used in efficient algorithms like Binary Search and Merge Sort.

---

# 4️ Problem Solving Mindset

Before writing code, always ask:

1. What is the input?
2. What is the output?
3. Can I solve it with brute force first?
4. Can sorting help?
5. Can searching help?
6. Can recursion reduce repeated work?

# 6️ Example Thought Process

Suppose you are given:

```text
[8, 3, 1, 7, 5]
```

And asked to find whether `7` exists.

### Method 1 — Linear Search

Check each element one by one.

### Method 2 — Sort + Search

Sort first:

```text
[1, 3, 5, 7, 8]
```

Then search more efficiently.

This is how problem solving improves with the right approach.
